Earth UI — Design Tokens

Linked from the same js/design-tokens.css that earth.html consumes. See DESIGN_TOKENS.md for usage rationale.

Color — accent families

Three accent tracks: cyan (primary chrome), amber (Storm Watch / cyclone), teal-mint (location panel hero). Each family ships hex + RGB triplet + a ladder of soft / border / border-hi / glow / bg-overlay variants so consumers can hit the right alpha without inlining a literal.

Color — status palette

Single source of truth for every layer-status pip. Stays put across themes — green=live, red=error is universal and re-mapping it would just confuse the convention.

Color — surface fills

The frosted-glass panel backgrounds. Layered radial-gradient overlays on top of these give each panel its characteristic hue without diverging the underlying value scale.

Color — text scale

Five-tier hierarchy. Skill-grid uses a slightly different mid-grey so refactors don't accidentally flatten the "this is data" vs "this is chrome" distinction.

Color — borders

Three weights for separators. Light theme uses higher alpha because dark separators on white have to be visibly darker than white separators on black to read at the same perceived weight.

Spacing rhythm

2 px-base scale; phones tighten 1-2 px per step. Mixing arbitrary px values is what tokens exist to prevent.

Border radii

Smaller affordances get smaller radii; hero panels get the loudest curve.
